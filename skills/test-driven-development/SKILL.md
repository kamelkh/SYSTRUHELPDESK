# test-driven-development

## Purpose
Use this skill for TDD-oriented work and controlled feature changes.

## What to do
- Define the expected behavior before coding.
- Write the smallest meaningful validation first.
- Keep the implementation minimal until the check passes.
- Prefer narrow, repeatable validation over broad assumptions.
- Refactor only after behavior is confirmed.

## Good output
- Clear expected behavior
- Small verification loop
- Minimal code changes
- Strong regression protection
