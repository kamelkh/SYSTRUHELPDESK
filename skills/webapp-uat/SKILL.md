# webapp-uat

## Purpose
Use this skill for browser-based validation of shipped web pages.

## What to do
- Check page title, hero, CTA links, form, responsiveness, and accessibility basics.
- Verify visible content matches the intended message.
- Look for broken layout, clipped content, or missing CTA behavior.
- Confirm the page works in the browser, not just in source code.
- Report concrete issues with reproduction steps.

## Good output
- Visual and functional findings
- Reproduction details
- Priority of issues
- Practical fixes
