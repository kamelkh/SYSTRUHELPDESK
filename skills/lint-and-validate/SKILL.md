# lint-and-validate

## Purpose
Use this skill after edits to verify the project still works.

## What to do
- Validate HTML structure and script syntax.
- Check responsive behavior and print styling.
- Confirm links, forms, and CTA actions work as expected.
- Prefer a focused validation step right after the first substantive edit.
- Fix the smallest issue first, then re-validate.

## Good output
- Clear validation result
- Failing file or line if relevant
- Minimal follow-up fix
