# create-pr

## Purpose
Use this skill when preparing a clean pull request or change summary.

## What to do
- Summarize only the material changes.
- Separate user-facing impact from internal details.
- Note validation performed and any residual risk.
- Keep the final message concise and reviewable.
- Prefer concrete file references over vague descriptions.

## Good output
- Short change summary
- Validation notes
- Risk notes
- Clear next steps
